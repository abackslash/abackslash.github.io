Android的activity的生命周期

在生命周期中只有Resumed、Paused和Stopped这三种状态是稳定的

Android启动模式

mode TaskID Activity ID Understand
standard 不变 改变 所有实例放在同一个栈，新建每个实例
singleTop 不变 当前activity处于栈顶不变，否则改变 所有实例放在同一个栈，当此activity处于栈顶不创建新实例，否则创建新的实例
singleTask 不变 不变 所有实例放在同一个栈，跳转到此activity时，弹出它上面的所有实例
singleInstance 变换activity时改变，否则不变 不变 此activity独享一个栈

LayoutInflater和findViewById()

LayoutInflater是用来查找layout下的xml布局文件并实例化的，而findViewById()是用来查找xml中具体widget控件的。

Activity和Fragment间对象传递

Android中，Activity和Fragment之间传递对象，可以通过将对象序列化并存入Bundle或者Intent中进行传递，也可以将对象转化为JSON字符串，进行传递。

序列化对象可以使用Java的Serializable的接口、Parcelable接口。转化成JSON字符串，可以使用Gson等库。
